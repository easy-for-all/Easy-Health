# Frontend Rules — Easy Health

## Stack
- Next.js
- React
- TypeScript
- Tailwind CSS

## Organização
- Usar estrutura por feature.
- Componentes compartilhados ficam em shared/components.
- Tipos compartilhados ficam em shared/types.

## UX
- Mobile-first.
- Poucos cliques.
- Botões claros.
- Estados visuais para loading, empty e error.
- Evitar telas densas.

## Código
- Componentes pequenos.
- Nomes claros.
- Evitar lógica de negócio no frontend.
- Validação de formulário no frontend apenas como apoio; validação real fica no backend.
